⚡ Triggers Implemented

trg_student_placed
AFTER INSERT on placements → Auto-sets student status to 'placed' and logs to audit
trg_no_double_placement
BEFORE INSERT on placements → Raises ABORT if student already placed (assertion-like)
trg_eligibility_check
BEFORE INSERT on applications → Enforces CGPA & backlogs eligibility criteria
trg_audit_student_update
AFTER UPDATE on students → Records every change to the audit_log table
trg_drive_seats_filled
AFTER INSERT on placements → Auto-completes drive when all seats are filled

🔒 Constraints & Assertions

✓CHECK — cgpa BETWEEN 0 AND 10, package > 0
✓UNIQUE — roll_no, email, (student+drive) pair
✓NOT NULL — all critical fields enforced
✓FOREIGN KEY — CASCADE DELETE, RESTRICT on dept
✓ENUM-like CHECK — status, tier, round columns
~Assertion via Trigger — No double placement (SQLite workaround)
~Assertion via Trigger — Eligibility criteria on application

🗃 CRUD Operations

CREATEStudents, Companies, Drives, Applications, Placements
READFiltered queries, JOINed views, aggregate analytics
UPDATEStudent profile, application round, drive status
DELETECascade-aware removal with FK constraints

📐 ER Design

Tables: departments, students, companies, drives, applications, placements, audit_log
Relationships:
• Student belongs to Department (M:1)
• Drive offered by Company (M:1)
• Student applies to Drive → Application (M:N)
• Student placed via Drive → Placement (1:1 for student)
• All mutations → audit_log (1:N)

📊 Normalization

1NF — All atomic values, no repeating groups
2NF — No partial dependencies (all non-key attrs depend on full PK)
3NF — No transitive dependencies (dept info not stored in students)
BCNF — Every determinant is a candidate key

Indexes: dept_id, status, student_id, drive_id, company_id, drive_date for query optimization
